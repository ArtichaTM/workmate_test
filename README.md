# Описание
Работа выполнялась с целью упрощения расширения функционала в сторону каких-то статистик или добавления новых `float|int|date` колонок.
- Колонка `name` (в csv `student`) стала обязательной как primary key и обязательно первая
- Приложение построен с помощью [uv](docs.astral.sh/uv), однако можно и воспользоваться pip
- Минимальный python 3.10 (можно было попробовать и ниже, функционал базовый)
- Используется архитектура пакетов python, поэтому возможна загрузка в pypi, Что даже проще через [uv](https://docs.astral.sh/uv/guides/package/)
- Используется библиотека `pre-commit` (устанавливается только при `uv sync --dev`) с flake8 и isort
- Возможно агрегировать данные по среднему, максимуму и минимуму используя `mean_`, `max_` и `min_` соответственно

## Примерные команды запуска
Работает "без" виртуального окружения (uv автоматически создаёт его и устанавливает python, если требуется)
```bash
uv run -m csv_printer --files temp/math.csv temp/physics.csv  # Запускает компиляцию отчётов в колонки по умолчанию `name, mean_coffee_spent`
```

Если необходимо работать через pip, пакет сначала нужно установить:
```bash
pip install .
```

И потом можно его запускать через python

```bash
python -m csv_printer --files temp/math.csv temp/physics.csv  # Аналог команды выше
```

И даже без `python -m`, если bin директория интерпретатора находится в PATH

```bash
csv_printer --files temp/math.csv temp/physics.csv  # Аналог команды выше
```

# Примечания
## Неодновременная сдача экзаменов
По исходным данным, все экзамены проходят все студенты одновременно, например:
```csv
student,date,coffee_spent,sleep_hours,study_hours,mood,exam
Мария Соколова,2024-06-01,100,8.0,3,отл,Математика
Мария Соколова,2024-06-02,120,8.5,2,отл,Математика
Мария Соколова,2024-06-03,150,7.5,4,отл,Математика
Павел Новиков,2024-06-01,380,5.0,10,норм,Математика
Павел Новиков,2024-06-02,420,4.5,11,устал,Математика
Павел Новиков,2024-06-03,470,4.0,13,устал,Математика
```

Ключ `2024-06-01-Математика` не обязательно должен присутствовать у каждого студента, то есть следующее также верно:

```csv
student,date,coffee_spent,sleep_hours,study_hours,mood,exam
Мария Соколова,2024-06-01,100,8.0,3,отл,Математика
Мария Соколова,2024-06-03,150,7.5,4,отл,Математика
Павел Новиков,2024-06-02,420,4.5,11,устал,Математика
```

Однако в библиотеке предполагается, что это может быть не так

## Обязательное колонка
Имя считается обязательной колонкой (*primary key*), которую нельзя исключить

## Указание обычных колонок при агрегации
Экзаменов много, а имя одно. Если указывается обычное поле, выдаётся предупреждение и колонка пропускается

## Не все версии корректные
Большинство коммитов на уровне разработке, приложение может не работать. Для проверки ссылайтесь на [версии](https://github.com/ArtichaTM/workmate_test/tags) (теги с версиями)


# Пример работы
Возьмём в пример следующий csv файл:
```csv
student,date,coffee_spent,sleep_hours,study_hours,mood,exam
Алексей Смирнов,2024-06-01,450,4.5,12,норм,Математика
Алексей Смирнов,2024-06-02,500,4.0,14,устал,Математика
Дарья Петрова,2024-06-01,200,7.0,6,отл,Математика
```

## Команда без аргументов
Команда без аргументов добавляет колонку mean_coffee_spent и сортирует по имени:

```console
$ csv_printer --files example.csv
+-----------------+-------------------+
|      name       | mean_coffee_spent |
+=================+===================+
| Алексей Смирнов |               475 |
+-----------------+-------------------+
|   Дарья Петрова |               200 |
+-----------------+-------------------+

$ csv_printer --files example.csv --report mean_coffee_spent --sort +name
+-----------------+-------------------+
|      name       | mean_coffee_spent |
+=================+===================+
| Алексей Смирнов |               475 |
+-----------------+-------------------+
|   Дарья Петрова |               200 |
+-----------------+-------------------+
```

## Несколько колонок
Можно запрашивать несколько колонок одновременно:

```console
$ csv_printer --files example.csv --report mean_sleep_hours max_coffee_spent
+-----------------+------------------+------------------+
|      name       | mean_sleep_hours | max_coffee_spent |
+=================+==================+==================+
| Алексей Смирнов |             4.25 |              500 |
+-----------------+------------------+------------------+
|   Дарья Петрова |              7.0 |              200 |
+-----------------+------------------+------------------+
```

## Сортировка
Сортировка осуществляется передачей колонки в аргументы `--sort`:

```console
$ csv_printer --files example.csv --sort mean_coffee_spent
+-----------------+-------------------+
|      name       | mean_coffee_spent |
+=================+===================+
| Алексей Смирнов |               475 |
+-----------------+-------------------+
|   Дарья Петрова |               200 |
+-----------------+-------------------+
```

Для обратной сортировки перед названием колонки поставьте `+`:

```console
$ csv_printer --files example.csv --sort +mean_coffee_spent
+-----------------+-------------------+
|      name       | mean_coffee_spent |
+=================+===================+
|   Дарья Петрова |               200 |
+-----------------+-------------------+
| Алексей Смирнов |               475 |
+-----------------+-------------------+
```

## Пример комплексной команды
Ниже представлен пример комплексной команды НЕ из примера:

```console
$ csv_printer --files temp/math.csv temp/physics.csv temp/programming.csv --report mean_coffe_spent mean_sleep_hours min_study_hours m_mood min_mod --sort min_study_hours
Колонка "mean_coffe_spent" основывается на колонке "coffe_spent", однако она не существует
Неизвестная колонка m_mood
Колонка "min_mod" основывается на колонке "mod", однако она не существует
+-------------------+------------------+-----------------+
|       name        | mean_sleep_hours | min_study_hours |
+===================+==================+=================+
|     Иван Кузнецов |              2.0 |              15 |
+-------------------+------------------+-----------------+
...
+-------------------+------------------+-----------------+
| Виктория Федорова |              8.0 |               2 |
+-------------------+------------------+-----------------+
```

Данная команда:
1. На основе файлов
   - *temp/math.csv*
   - *temp/physics.csv*
   - *temp/programming.csv*
2. Создаёт таблицу из колонок
   - *mean_coffe_spent* - опечатка (`coffe`), сообщение об ошибке есть
   - *mean_sleep_hours* - добавляется второй колонкой
   - *min_study_hours* - добавляется третьей колонкой
   - *m_mood* - ошибка, колонки не существует
   - *min_mod* - ошибка, базовой колонки (после `min_`) не существует
3. И сортирует её по колонке *min_study_hours* в порядке убывания
